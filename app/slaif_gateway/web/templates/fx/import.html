Preview Policy

This workflow parses and validates uploaded content in memory only. It does not call external FX services, does not call providers, does not store uploaded content, and does not write audit rows or fx_rates rows.

Accepted fields: base_currency, quote_currency, rate, source, valid_from, valid_until, metadata, notes. Decimal rates must be strings. Secret-looking source, note, or metadata values are rejected.

Limits: {{ settings.FX_IMPORT_MAX_ROWS }} rows and {{ settings.FX_IMPORT_MAX_BYTES }} bytes.

Input

Use either upload or pasted content, not both.

Execute Import

This will create FX rows after the server re-parses and re-validates the submitted CSV or JSON. It is all-or-nothing: invalid, duplicate, update, or conflict rows block the entire import.

Execution does not trust preview HTML, hidden normalized rows, or client-side classification. External FX APIs and providers are not called, and uploaded raw content is not stored.
